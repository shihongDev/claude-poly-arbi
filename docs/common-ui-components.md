# Common UI Components Reference

A catalog of widely-used UI components found across modern design systems, sourced from [Component Gallery](https://component.gallery/components/).

---

## Form Controls

| Component | Description |
|-----------|-------------|
| **Button** | Triggers an action or event (submit, cancel, navigate). The most fundamental interactive element. |
| **Button group** | Groups related buttons together as a single visual unit (e.g., toolbar actions, toggle options). |
| **Checkbox** | Allows users to select one or more options from a set. Supports checked, unchecked, and indeterminate states. |
| **Color picker** | Lets users select a color value via a visual palette or text input. |
| **Combobox** | Combines a text input with a dropdown list, allowing users to type to filter or select from predefined options. |
| **Date input** | A specialized text input for entering dates, often with format masking. |
| **Datepicker** | A calendar-based control for selecting dates visually. |
| **Fieldset** | Groups related form fields together with an optional legend/label. |
| **File upload** | Allows users to select and upload files from their device (drag-and-drop or browse). |
| **Form** | A container that groups input elements and handles submission of user data. |
| **Label** | Associates descriptive text with a form control for accessibility and usability. |
| **Radio button** | Allows users to select exactly one option from a mutually exclusive set. |
| **Rating** | Lets users provide a score (typically 1-5 stars) for an item. |
| **Rich text editor** | A WYSIWYG editor for formatting text with bold, italic, lists, links, etc. |
| **Search input** | A text input optimized for search queries, often with a clear button and search icon. |
| **Segmented control** | A set of two or more segments that function as mutually exclusive buttons (similar to tabs but for values). |
| **Select** | A dropdown control that lets users choose one option from a predefined list. |
| **Slider** | Lets users select a value or range from a continuous or discrete scale by dragging a handle. |
| **Text input** | A single-line field for entering short-form text (names, emails, etc.). |
| **Textarea** | A multi-line field for entering longer-form text (comments, descriptions). |
| **Toggle** | A switch control that represents an on/off binary state. |

## Navigation

| Component | Description |
|-----------|-------------|
| **Breadcrumbs** | Shows the user's current location within a site hierarchy, with links back to parent pages. |
| **Footer** | The bottom section of a page, typically containing links, copyright info, and secondary navigation. |
| **Header** | The top section of a page, typically containing logo, primary navigation, and global actions. |
| **Link** | A clickable text or element that navigates the user to another page or resource. |
| **Navigation** | A collection of links that provides the primary or secondary wayfinding structure of a site/app. |
| **Pagination** | Controls for navigating between pages of content (previous/next, page numbers). |
| **Skip link** | A hidden-until-focused link that allows keyboard users to skip repetitive content (e.g., nav) and jump to main content. |
| **Stepper** | Displays progress through a multi-step process (e.g., checkout flow, onboarding wizard). |
| **Tabs** | Organizes content into multiple panels, only one of which is visible at a time. |

---

## Data Display

| Component | Description |
|-----------|-------------|
| **Avatar** | A visual representation of a user or entity, typically a photo, initials, or icon. |
| **Badge** | A small status indicator attached to another element, showing counts, labels, or states. |
| **Card** | A contained, flexible surface for displaying grouped content and actions about a single subject. |
| **Heading** | A typographic element that introduces sections of content (h1-h6). |
| **Icon** | A small graphical symbol representing an action, object, or concept. |
| **Image** | Displays visual media with optional alt text, captions, and responsive behavior. |
| **List** | An ordered or unordered collection of related items displayed vertically. |
| **Quote** | Displays a quotation with optional attribution, visually distinct from surrounding text. |
| **Table** | Organizes data into rows and columns for comparison and scanning. |
| **Tree view** | Displays hierarchical data as an expandable/collapsible nested list. |
| **Video** | Embeds video content with playback controls. |

---

## Feedback & Status

| Component | Description |
|-----------|-------------|
| **Alert** | A prominent message that communicates important information, warnings, errors, or success states. |
| **Empty state** | Placeholder content shown when a view has no data to display, guiding users on next steps. |
| **Progress bar** | A horizontal bar that indicates the completion percentage of an operation. |
| **Progress indicator** | A general indicator (determinate or indeterminate) showing that an operation is in progress. |
| **Skeleton** | A placeholder that mimics the shape of content while it loads, reducing perceived wait time. |
| **Spinner** | An animated indicator showing that content or an action is loading. |
| **Toast** | A brief, auto-dismissing notification that appears temporarily (usually at screen edges). |

---

## Overlays & Popovers

| Component | Description |
|-----------|-------------|
| **Drawer** | A panel that slides in from the edge of the screen, used for secondary content or navigation. |
| **Dropdown menu** | A list of actions or options that appears when a trigger element is activated. |
| **Modal** | A dialog that overlays the page, requiring user interaction before returning to the main content. |
| **Popover** | A floating container triggered by a click, showing contextual content positioned near its trigger. |
| **Tooltip** | A small floating label that appears on hover/focus to provide additional context about an element. |

---

## Layout & Structure

| Component | Description |
|-----------|-------------|
| **Accordion** | A vertically stacked set of collapsible sections, allowing users to expand/collapse content. |
| **Carousel** | A horizontally scrollable set of items (images, cards) with navigation controls. |
| **Hero** | A large, prominent banner section typically at the top of a page, featuring a headline, description, and CTA. |
| **Separator** | A visual divider (horizontal rule) that separates sections of content. |
| **Stack** | A layout utility that spaces child elements evenly along a vertical or horizontal axis. |
| **Visually hidden** | A utility that hides content visually while keeping it accessible to screen readers. |

---

## Component by File Type

| Component | Description |
|-----------|-------------|
| **File** | Displays information about a file (name, type, size, preview). |
